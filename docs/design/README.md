# Design Docs

This folder holds forward-looking design and proposal documents.

Use these for:
- implementation planning
- tradeoff review
- design intent before code exists

Do not use these as proof that something is already built or deployed.

If a design ships:
- update `../STATE.md` for current status
- update `../LOYALTY_PLATFORM_ESSENTIALS.md` if it creates a durable rule
- update `../LOYALTY_PLATFORM_MASTER.md` if it changes architecture or subsystem behavior
