# Session History Index

Purpose: track historical session handoffs and retrospectives without letting
them compete with the live startup path.

Start with `START_HERE.md`, not this file.

Routine continuity now belongs in `../ACTIVE_WORK.md`, not in new versioned
`HANDOFF_FROM_*` files.

## Use These For

- understanding why a past decision was made
- reconstructing the scope of a specific session
- postmortem / trust / process analysis
- tracing the history of a refactor

## Do Not Use These For

- the first startup path in a new session
- current deploy state
- current test/deploy workflow
- canonical platform rules

Those live in:
- `../START_HERE.md`
- `../HANDOFF.md`
- `../STATE.md`
- `../WORKFLOWS.md`
- `BEFORE_YOU_WRITE.md`
- `LOYALTY_PLATFORM_ESSENTIALS.md`
- `LOYALTY_PLATFORM_MASTER.md`
