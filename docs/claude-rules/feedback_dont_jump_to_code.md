---
name: Don't jump to coding
description: Wait for Bill to finish discussing before writing any code - discuss first, code later
type: feedback
---

Do NOT start coding the moment Bill mentions a task. He often wants to discuss, think through, and formulate the approach before any code is written.

**Why:** Bill has seen repeated sessions where Claude jumps straight into writing code before the requirements are clear, leading to wasted work, wrong approaches, and meltdowns when things have to be redone.

**How to apply:** When Bill brings up a topic, LISTEN and DISCUSS. Ask clarifying questions. Talk through the approach. Only start writing code when Bill explicitly says to go ahead or it's unmistakably clear he wants implementation. Default to conversation, not action.
